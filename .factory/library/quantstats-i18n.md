# QuantStats Tearsheet i18n Architecture

## Key Insight

QuantStats generates most metric labels **dynamically in Python** (in `quantstats/reports.py`), not from the HTML template. The HTML template only controls ~3 static strings (`Key Performance Metrics`, `Worst 10 Drawdowns`, `Generated by`).

## Translation Approach

A two-pronged approach is required for i18n:

1. **Custom HTML template** (`src/reporting/templates/tearsheet_zh.html`): Handles the few static HTML strings in the template.
2. **Post-processing function** (`_translate_metrics_to_chinese()` in `src/reporting/tearsheet.py`): Does string replacement on the final HTML output for 100+ dynamically generated metric labels.

The post-processing uses two replacement patterns:
- `>{english}<` — Matches labels inside HTML tags (primary, most reliable)
- ` {english} ` — Catches labels in free text (secondary fallback)

## Ordering Matters

The translation dict is iterated in insertion order. Longer compound terms (e.g., "Rolling Sharpe") must appear before shorter substrings (e.g., "Sharpe") to avoid partial matches.

## Coverage Gap

~25 less-common QuantStats metrics remain untranslated in Chinese mode. Core metrics (Sharpe, Sortino, CAGR, Max Drawdown, Volatility, etc.) and all section headers are translated.
